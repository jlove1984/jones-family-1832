**Existing Logo Elements:**

- **Continental Reference:** Features Chipata, Florida (African heritage connection)
- **Founding Year:** "1832" prominently displayed
- **Family Name:** "JONES FAMILY" at top
- **Reunion Theme:** "REUNION" text at bottom
- **Heritage Symbols:** Decorative elements representing family legacy
- **Motto Elements:** "John - Selina - Mary" and "Maggie - Wade II - Semaj" references

### 2.2 Logo Usage for Digital Platform

#### **Primary Logo (Full Crest)**

- Usage: Website header, official documents, print materials, hero sections
- Minimum size: 120px width
- Format: Complete family crest as provided

#### **Simplified Logo**

- Usage: Navigation bar, mobile header, email signatures
- Minimum size: 80px width
- Format: Central shield/crest portion only

#### **Icon Mark**

- Usage: Favicon, app icon, social media avatar
- Minimum size: 32px × 32px
- Format: Shield center or "JF" monogram derived from crest

#### **Theme Variations**

- **Light Theme:** Full color logo on light backgrounds
- **Dark Theme:** White/inverted version for dark backgrounds
- **High Contrast:** Monochrome versions for accessibility

### 2.3 Logo Usage Guidelines

**Clear Space:**

- Maintain minimum clear space equal to height of "1832" text on all sides
- No other graphic elements within clear space

**Prohibited Uses:**

- ❌ Do not stretch or distort proportions
- ❌ Do not rotate logo
- ❌ Do not apply drop shadows or effects
- ❌ Do not change colors outside approved palette
- ❌ Do not place on busy backgrounds without sufficient contrast
- ❌ Do not recreate or modify logo elements

**Backgrounds:**

- ✅ Primary logo on white, cream, or light neutral backgrounds
- ✅ White logo on primary brand colors or dark backgrounds
- ✅ Ensure minimum 4.5:1 contrast ratio

### 3.6 Theme Switching

**Default Behavior:**

- Respect system preference (`prefers-color-scheme`)
- User can manually override system preference
- Store preference in localStorage or user account settings
- Smooth transition animation (200ms) when switching themes

**Toggle Location:**

- Header navigation (sun/moon icon)
- User account settings page
- Keyboard shortcut: `Ctrl/Cmd + Shift + D`

### 3.7 Color Accessibility

**WCAG 2.1 AA Compliance:**

- Text on background: Minimum 4.5:1 contrast ratio
- Large text (18pt+): Minimum 3:1 contrast ratio
- UI components: Minimum 3:1 contrast ratio

**Approved Combinations (Light Theme):**

- ✅ Charcoal text (#2C2C2C) on White (#FFFFFF) = 15.3:1
- ✅ Heritage Green (#2D5016) on Cream (#F5F1E8) = 8.2:1
- ✅ White (#FFFFFF) on Heritage Green (#2D5016) = 10.5:1
- ✅ Charcoal (#2C2C2C) on Cream (#F5F1E8) = 13.8:1

**Approved Combinations (Dark Theme):**

- ✅ Light Text (#E5E5E5) on Rich Black (#0F0F0F) = 14.8:1
- ✅ Light Text (#E5E5E5) on Dark Gray (#1A1A1A) = 13.2:1
- ✅ Heritage Green Light (#4A8028) on Rich Black (#0F0F0F) = 7.1:1
- ✅ Legacy Gold Bright (#D4B896) on Rich Black (#0F0F0F) = 9.5:1

## 4. ✍️ Typography

### 4.1 Font Family System

#### **Primary Typeface: Playfair Display (Serif)**

- **Usage:** Headings, hero text, logo, elegant emphasis
- **Weights:** Regular (400), SemiBold (600), Bold (700)
- **Character:** Classic, elegant, timeless, heritage-focused
- **Source:** Google Fonts
- **Fallback:** Georgia, 'Times New Roman', serif

#### **Secondary Typeface: Inter (Sans-serif)**

- **Usage:** Body text, UI elements, navigation, forms
- **Weights:** Regular (400), Medium (500), SemiBold (600), Bold (700)
- **Character:** Modern, readable, neutral, accessible
- **Source:** Google Fonts
- **Fallback:** -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif

#### **Monospace Typeface: Fira Code**

- **Usage:** Code snippets, technical documentation
- **Weight:** Regular (400)
- **Source:** Google Fonts
- **Fallback:** 'Courier New', monospace

### 4.2 Type Scale

| Element | Font | Size | Weight | Line Height | Letter Spacing |
| --- | --- | --- | --- | --- | --- |
| **Hero Title** | Playfair Display | 64px / 4rem | 700 | 1.1 | -0.02em |
| **H1** | Playfair Display | 48px / 3rem | 700 | 1.2 | -0.01em |
| **H2** | Playfair Display | 36px / 2.25rem | 600 | 1.3 | 0 |
| **H3** | Playfair Display | 28px / 1.75rem | 600 | 1.4 | 0 |
| **H4** | Inter | 20px / 1.25rem | 600 | 1.5 | 0 |
| **Body Large** | Inter | 18px / 1.125rem | 400 | 1.6 | 0 |
| **Body Regular** | Inter | 16px / 1rem | 400 | 1.6 | 0 |
| **Body Small** | Inter | 14px / 0.875rem | 400 | 1.5 | 0 |
| **Caption** | Inter | 12px / 0.75rem | 400 | 1.4 | 0.01em |
| **Button** | Inter | 16px / 1rem | 600 | 1 | 0.02em |

### 4.3 Responsive Typography

**Mobile (< 768px):**

- Hero Title: 40px
- H1: 32px
- H2: 24px
- H3: 20px
- Body: 16px (unchanged)

**Tablet (768px - 1023px):**

- Hero Title: 52px
- H1: 40px
- H2: 30px
- H3: 24px
- Body: 16px (unchanged)

### 4.4 Typography Guidelines

**Hierarchy:**

- Use Playfair Display for major headings to establish visual hierarchy
- Use Inter for body text and UI elements for optimal readability
- Limit heading levels to H1-H4 per page for clarity

**Emphasis:**

- **Bold** for strong emphasis
- *Italic* for gentle emphasis or citation
- Underline reserved for links only

**Line Length:**

- Optimal: 50-75 characters per line
- Maximum: 90 characters per line
- Minimum: 30 characters per line

### 5.15 Loading, Empty, and Error States

**Loading:** Use skeleton placeholders (rectangles/circles in Light Gray/Dark Gray) matching content shape (cards, list rows), or spinner (Legacy Gold or Heritage Green). Prefer skeletons for list/grid; spinner for buttons or inline. Minimum size for spinner: 24px.

**Empty state:** Centered block: optional illustration or icon (e.g. 48px), heading (Inter SemiBold 18px), short copy (Inter Regular 14px), primary CTA button when applicable (e.g. "Add first achievement", "Upload photos"). Use Muted Text for copy. Apply to Directory, Achievements, Gallery, and other list views.

**Error / edge:**
- **404:** Full-page or section: "Page not found" (Playfair Display or Inter), short message, link back to Home. Use Standard Card or centered layout.
- **Auth gate:** When unauthenticated user hits protected page (e.g. Directory): message "Log in to view the family directory" (or similar), primary CTA "Log in", secondary "Go to homepage". Same card/centered pattern.
- **Session expired:** Inline or modal: "Your session has expired. Please log in again." CTA: "Log in".
- **Form validation:** Per 5.2 (error border, helper text). Inline errors per field; do not rely only on toast.

## 9. ♿ Accessibility Guidelines

### 9.1 WCAG 2.1 AA Compliance

**Color Contrast:**

- ✅ Text: Minimum 4.5:1 ratio
- ✅ Large text: Minimum 3:1 ratio
- ✅ UI components: Minimum 3:1 ratio
- ✅ All approved color combinations tested

**Keyboard Navigation:**

- All interactive elements reachable via Tab
- Clear focus indicators (2px outline)
- Logical tab order
- Skip links for main content

**Screen Readers:**

- Semantic HTML elements
- Proper ARIA labels on custom components
- Alt text for all meaningful images
- Proper heading hierarchy

**Motion & Animation:**

- Respect prefers-reduced-motion
- No flashing content
- Animations optional, never required

### 9.2 Focus and Keyboard (Lightbox, Modal, Calendar)

**Lightbox / modal:** Focus trap: keep tab order within the dialog; no tab to background. On open: move focus to first focusable element (e.g. close button or image). On close: return focus to trigger element. Keyboard: Escape closes; Arrow Left/Right for prev/next in lightbox. Use aria-modal="true", role="dialog", and appropriate aria-label.

**Birthday calendar:** Month/year selector must be focusable and operable by keyboard (Tab to control; Enter/Space to activate; arrows for prev/next month if applicable). Day cells: focusable; Enter/Space to open popover. Popover: focus trap; Escape closes. Screen reader: announce current month/year; announce day and "N birthdays" when applicable.

### 9.3 Responsive Behavior (Directory, Complex Forms)

**Directory:** Desktop: use table (5.7). Tablet/mobile: switch to **card list** — one card per member with primary fields (name, branch, city/state) and "View profile" link. Avoid horizontal scroll on small screens; cards stack vertically.

**Complex forms (RSVP, achievement submission, profile edit):** Mobile-first: single column; group related fields; sufficient touch targets (min 44px). Optional: multi-step for very long forms (e.g. step indicator, Next/Back). Keep primary CTA visible (sticky or above fold).

### 9.4 Inclusive Design

**Age Consideration:**

- Larger touch targets (minimum 44px × 44px)
- Clear, readable text sizes
- Simple, straightforward interactions
- Forgiving error handling

**Technical Ability:**

- Progressive enhancement
- Graceful degradation
- Clear labeling and instructions
- Helpful error messages

### 10.3 Theme Implementation

**React/Next.js Example:**

```tsx
// app/providers/theme-provider.tsx
import { createContext, useContext, useEffect, useState } from 'react'

type Theme = 'light' | 'dark' | 'system'

const ThemeContext = createContext<{
  theme: Theme
  setTheme: (theme: Theme) => void
}>({
  theme: 'system',
  setTheme: () => null,
})

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState<Theme>('system')

  useEffect(() => {
    const root = window.document.documentElement
    root.classList.remove('light', 'dark')

    if (theme === 'system') {
      const systemTheme = window.matchMedia('(prefers-color-scheme: dark)').matches
        ? 'dark'
        : 'light'
      root.classList.add(systemTheme)
    } else {
      root.classList.add(theme)
    }
  }, [theme])

  return (
    <ThemeContext.Provider value= theme, setTheme >
      {children}
    </ThemeContext.Provider>
  )
}

export const useTheme = () => useContext(ThemeContext)
```

## 13. Future State & Roadmap Alignment

The Technical Design Document (Section 11: Future Roadmap Phase 2–3) defines the following post-launch directions. This section records UX and design-system considerations so the current design can accommodate future features without breaking consistency.

**Reference:** [Technical Design Document](C:\Dev\jones-family-hub_executive-summary\📘 Technical Design Document (1) 2eedfd535e21805fa274e2a0c9fccb1b.md) — Section 11. Future Roadmap (Phase 2-3).

### 13.1 Live-Stream Reunion Capability

**UX considerations:** Dedicated video player (or embed) with play/pause, volume, fullscreen; optional live chat or Q&A panel. Reuse light theme/dark theme and Standard Card or Feature Card for container. Consider: placement on reunion page vs separate /reunion/live route; accessibility (captions, reduced motion); mobile layout (stack player above content). Icon: use Play/Pause, Volume, Maximize from action set (7.3).

### 13.2 Family Merchandise Store

**UX considerations:** Product grid (reuse gallery-style card grid); product detail (image, description, size/variant selector); cart (drawer or page); checkout flow. Phase 2 payments (PayPal, optional CashApp) may add a payment-method selector — extend payment tier selector pattern (5.13) or add radio/card choice for payment provider. Ensure store fits existing nav (e.g. "Store" link); use Legacy Gold or Terracotta for commerce CTAs if desired to distinguish from reunion dues.

### 13.3 Interactive Genealogy Tree Explorer

**UX considerations:** Interactive tree visualization (nodes = people; edges = relationships). Design node style (photo, name, dates) to align with directory cards and typography. Zoom/pan controls; mobile: consider simplified view or horizontal scroll. Color: Heritage Green for lines/connectors; nodes use Standard Card or compact variant. Accessibility: keyboard navigation between nodes; screen reader summary of lineage path.

### 13.4 AI-Powered Photo Tagging

**UX considerations:** Tagging UI in gallery (select photo → add/edit tags or faces); search/filter by tag; admin moderation queue if tags are user- or AI-suggested. Reuse filter and chip patterns (5.16); ensure tag input matches form elements (5.2). Consider lightbox or inline tag editor; keep WCAG 2.1 AA for any new controls.

### 13.5 React Native Mobile App

**UX considerations:** Design tokens (colors, typography, spacing) and component patterns in this document are intended to be portable. Use the same semantic tokens (Heritage Green, Legacy Gold, etc.) and type scale in the mobile app for brand consistency. Touch targets already specified (min 44px) support mobile. Document token export (e.g. Tailwind → React Native or shared JSON) in Implementation Notes when the app project starts. Logo and icon mark (2.2) apply to app icon and splash.

### 13.6 Elder Voice Recordings Archive

**UX considerations:** Audio player component (play/pause, progress, volume, optional playback speed); playlist or list view (title, duration, date, speaker). Use Standard Card for each recording; Legacy Gold or Terracotta for play accent. Consider timeline or category filters similar to achievements. Accessibility: ensure player is keyboard operable and screen-reader friendly; provide transcript link if available.

### 13.7 Phase 2 Payments (PayPal, CashApp)

**UX considerations:** Payment tier selector (5.13) and checkout flow will extend to multiple providers. Add a **payment method** step or selector (e.g. Stripe, PayPal, CashApp QR) before or alongside tier selection; reuse card/radio pattern. Success/failure screens (5.15) remain; ensure receipt or confirmation shows method used. No visual change to brand; only flow and copy additions.
